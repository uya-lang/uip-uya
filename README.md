# uIP-uya

使用 **uya** 从零重写 **C 语言版 uIP 协议栈**。

uIP-uya 是一个面向资源受限系统编程的实验性项目，目标不是重新设计协议栈，而是尽量忠实保留 C 版 uIP 的核心实现方式，并用 uya 重新表达出来。

## 项目简介

本项目关注的是 **C 版 uIP 的语义级重写**，重点包括：

- 单一主缓冲区 `uip_buf`
- 以 `uip_process(flag)` 为中心的主处理流程
- 固定容量 TCP / UDP / ARP 状态表
- 编译期配置裁剪
- 驱动与协议栈紧密协作的调用模型

也就是说，本项目不是“参考 uIP 思想做一个新协议栈”，而是希望把 **uIP 本身** 用 uya 重写一遍。

## 设计方向

### 忠实重写

尽量保留 C 版 uIP 的真实组织方式，而不是先做现代化抽象再实现。

### 零 GC

继续采用静态分配、固定容量、可预测资源占用的实现路线。

### 可测试演进

在不背离原版架构的前提下，为输入输出行为、状态迁移与协议路径建立测试基线。

## 预期范围

随着项目推进，计划逐步覆盖：

- `uip.c` 对应的核心处理逻辑
- `uip.h` 对应的数据结构与接口
- `uip_arp.c` 对应的 ARP / Ethernet 处理
- `uipopt.h` / `uip-conf.h` 风格配置模型
- `uiplib.c` 风格辅助函数
